1. Introduo

Neste trabalho iremos analisar os dados de commits no Github. Para tal, recolhemos uma amostra de dados da atividade global do Github. Os dados brutos contm a contagem de quantas pessoas editaram arquivos com cada extenso em cada dia de 2016 e 2017.

Nosso principal objetivo responder as seguintes perguntas:

  • 1) Para cada linguagem de programao, existe uma diferena significativa na sua popularidade durante a semana e durante o fim de semana?

  • 2) Existe uma diferena significativa na variao da popularidade das duas linguagens de programao quando voc compara o quanto a popularidade de cada uma varia durante os dias da semana?

1.1 Tratamento dos dados

importante para esta anlise ter conhecimento a respeito da data do commit e o dia da semana que corresponde referida data. Portanto, utilizamos a biblioteca lubridate para obter estes dados, e armazenamos estas informaes em novas colunas chamadas data e dia_semana.

dados_github = dados_github %>% 
  mutate(data = lubridate::ymd(paste(the_year, the_month, month_day)), 
         dia_semana = lubridate::wday(data))

2. Respondendo as questes

1) Para cada linguagem de programao, existe uma diferena significativa na sua popularidade durante a semana e durante o fim de semana?

Inicialmente, realizamos algumas manipulaes nos dados para que nossa anlise se torne mais objetiva e de fcil entendimento. Portanto, separamos os dados em 4(quatro) grupos:

  • commits em Java no fim de semana (github_java_fds)

  • commits em Java durante a semana (github_java_sem)

  • commits em Python no fim de semana (github_python_fds)

  • commits em Python durante a semana (github_python_sem)

Alm disso, criamos uma nova coluna (isFDS) de valor booleando (True ou False), onde o valor ser verdadeiro se o dia do commit no fim de semana, ou ser falso caso contrrio.

# Linguagem: Java (java)

github_java = dados_github %>% 
  filter(file_extension == "java") %>%
  mutate(isFDS = dia_semana > 5)

github_java_fds = github_java %>% 
  filter(dia_semana > 5)

github_java_sem = github_java %>%
  filter(dia_semana < 6)
# Linguagem: Python (py)

github_python = dados_github %>% 
  filter(file_extension == "py") %>%
  mutate(isFDS = dia_semana > 5)

github_python_fds = github_python %>% 
  filter(dia_semana > 5)

github_python_sem = github_python %>%
  filter(dia_semana < 6)

Nesse ponto, necessrio calcular os intervalos de confiana da mdia da popularidade para as linguagens de programao Java e Python. Dessa forma, ser possvel comparar a mdia da popularidade para os 4(quatro) grupos os quais estamos interessados em analisar. Alm disso, plotaremos um grfico que representa estes intervalos de confiana.

# IC para Java no final de semana
java_mean.fds =  bootstrap(github_java_fds, mean(users)) %>% 
  CI.bca(probs = c(.025, .975))

# IC para Java durante a semana
java_mean.sem = bootstrap(github_java_sem, mean(users)) %>%
  CI.bca(probs = c(.025, .975))

# IC para Python no final de semana
python_mean.fds = bootstrap(github_python_fds, mean(users)) %>%
  CI.bca(probs = c(.025, .975))

# IC para Python durante a semana
python_mean.sem = bootstrap(github_python_sem, mean(users)) %>%
  CI.bca(probs = c(.025, .975))

df = data.frame(rbind(java_mean.fds, 
                      java_mean.sem,
                      python_mean.fds,
                      python_mean.sem))

df$`mdia` = c("Java Final de Semana", "Java Semana", "Python Final de Semana", "Python Semana")

df %>% 
  ggplot(aes(x = `mdia`, ymin = X2.5., ymax = X97.5.)) + 
  geom_errorbar(width = .2)

Dado o grfico acima, podemos observar que aparentemente existem diferenas significativas de popularidade durante a semana e no fim de semana para as duas linguagens de programao. No entanto, este grfico no pode nos dar a certeza dessa afirmao, necessrio quantificar essa diferena para saber se realmente significativa.

Utilizando a biblioteca bootstrap2, podemos calcular a diferena dos intervalos de confiana para as mdias de popularidade. Dessa forma, possvel concluir se a diferena significativa. O grfico a seguir calcula a diferenas dos intervalos de confiana para a linguagem de programao Java.

# Java
b.diff.means = bootstrap2(github_java$users, 
                          treatment = github_java$isFDS, 
                          mean)

means.diff = CI.percentile(b.diff.means, probs = c(.05, .95))

data.frame(means.diff) %>% 
  ggplot(aes(x = "Diferena", ymin = X5., ymax = X95.)) + 
  geom_errorbar(width = .2) + 
  geom_hline(yintercept = 0, colour = "darkorange")

De acordo com o grfico acima, existe sim diferena significativa na popularidade durante a semana e o fim de semana para linguagem Java, tal que o valor 0(zero) no pertence ao intervalo de confiana da diferena. Portanto, podemos concluir que a mdia de commits da linguagem Java maior durante a semana do que no fim de semana. Precisamos agora plotar o mesmo grfico para a linguagem de programao Python.

# Python
b.diff.means = bootstrap2(github_python$users, 
                          treatment = github_python$isFDS, 
                          mean)

means.diff = CI.percentile(b.diff.means, probs = c(.05, .95))

data.frame(means.diff) %>% 
  ggplot(aes(x = "Diferena", ymin = X5., ymax = X95.)) + 
  geom_errorbar(width = .2) + 
  geom_hline(yintercept = 0, colour = "darkorange")

De acordo com o grfico acima, existe sim diferena significativa na popularidade durante a semana e o fim de semana para a linguagem de programao Python, tal que o valor 0(zero) no pertence ao intervalo de confiana da diferena. Portanto, podemos concluir que a mdia de commits da linguagem Python maior durante a semana do que no fim de semana.

Respondendo pergunta (1): Sim, existe diferena significativa na popularidade entre commits durante a semana e no fim de semana para as duas linguagens de programao selecionadas.

2) Existe uma diferena significativa na variao da popularidade das duas linguagens de programao quando voc compara o quanto a popularidade de cada uma varia durante os dias da semana?

Para esta questo, precisamos selecionar todos os commit feitos nas linguagens Java e Python durante a semana. Atribumos estes dados a varivel (github_py_java).

github_py_java = dados_github %>%
  filter(file_extension == "java" 
         | file_extension == "py", 
         dia_semana < 6)

Dessa vez, estamos interessados em analisar a variao da popularidade, ento podemos calcular intervalos de confiana utilizando o desvio padro da popularidade nas linguagens de programao Java e Python, apenas para os commits realizados durante a semana. Plotaremos um grfico que representa adequadamente estes intervalos de variao.

# IC para Java durante a semana
java_sd.sem = bootstrap(github_java_sem, sd(users)) %>%
  CI.bca(probs = c(.025, .975))

# IC para Python durante a semana
python_sd.sem = bootstrap(github_python_sem, sd(users)) %>%
  CI.bca(probs = c(.025, .975))

df = data.frame(rbind(java_sd.sem,
                      python_sd.sem))

df$`varincia` = c("Java Semana", "Python Semana")

df %>% 
  ggplot(aes(x = `varincia`, ymin = X2.5., ymax = X97.5.)) + 
  geom_errorbar(width = .2)

Aparentemente, existe uma diferena significativa na variao da popularidade entre Java e Python durante a semana. No entanto, este grfico no pode nos dar a certeza dessa afirmao, necessrio quantificar essa diferena para saber se realmente significativa.

# Diferena das variaes
b.diff.dp = bootstrap2(github_py_java$users, 
                          treatment = github_py_java$file_extension, 
                          sd)

dp.diff = CI.percentile(b.diff.dp, probs = c(.05, .95))

data.frame(dp.diff) %>% 
  ggplot(aes(x = "Diferena", ymin = X5., ymax = X95.)) + 
  geom_errorbar(width = .2) + 
  geom_hline(yintercept = 0, colour = "darkorange")

De acordo com o grfico acima, existe sim diferena significativa na variao da popularidade durante a semana entre as linguagens de programao Java e Python, tal que o valor 0(zero) no pertence ao intervalo de confiana da diferena. Portanto, podemos concluir que a variao do nmero commits da linguagem Python maior do que os commits para a linguagem Java durante a semana.

Respondendo pergunta (2): Sim, existe diferena significativa na variao da popularidade das linguagens de programao Java e Python para os commits realizados durante a semana.
